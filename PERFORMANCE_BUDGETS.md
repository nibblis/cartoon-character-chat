# Перформанс-бюджеты и Мониторинг

Этот проект придерживается строгих ограничений производительности для обеспечения плавного UX на широком спектре устройств (от Low-end до Flagship).

---

## 1. Дашборд метрик и пороговые значения (Budgets)

Мы отслеживаем следующие ключевые показатели (KPI):

| Метрика | Целевое значение (Budget) | Критическое (Alert) | Инструмент |
| :--- | :--- | :--- | :--- |
| **RAM (Memory)** | < 150 MB (Normal state) | > 250 MB | Android Profiler / LeakCanary |
| **CPU Usage** | < 10% (Idle) | > 30% (Idle) | Android Profiler |
| **GPU (Frame Rate)** | 60 FPS (16.6ms / frame) | < 55 FPS | Profile HWUI Rendering |
| **TTF (Time To First)** | < 500ms (Cold Start) | > 1.2s | Macrobenchmark |
| **Binary Size** | < 15 MB (AAB) | > 20 MB | APK Analyzer |

---

## 2. Настройка Алертов и Автоматизация

1.  **Macrobenchmark Tests:** В проекте настроены тесты для замера скорости запуска и плавности скролла списков. При превышении порога (например, запуск > 1.5 сек) билд в CI помечается как `Unstable`.
2.  **Strict Mode:** В дебаг-сборках включен `StrictMode`, который алертит (крашит приложение или логирует) при сетевых запросах или операциях с диском в Main thread.
3.  **LeakCanary:** Интегрирован для автоматического обнаружения утечек памяти в реальном времени во время тестирования.

---

## 3. Действия при нарушениях (Incident Response)

Если метрика выходит за пределы бюджета, предпринимаются следующие шаги:

### А. RAM Leak (> 250 MB)
*   Анализ Heap Dump через Memory Profiler.
*   Проверка жизненного цикла тяжелых объектов и подписок (Flow/Listeners).

### Б. Frame Drop / Jank (< 55 FPS)
*   Использование **System Trace** для поиска "длинных" кадров.
*   Оптимизация Compose-рекомпозиций (использование `derivedStateOf`, `remember`).
*   Проверка работы GC (см. `GC_TUNING_AND_LOG_ANALYSIS.md`).

### В. Медленный запуск (TTF > 1.2s)
*   Проверка графа зависимостей (Koin): переход на `lazy` инициализацию.
*   Использование `Baseline Profiles` для предварительной компиляции критических путей.

---

**Итог:** Перформанс-бюджеты — это не просто цифры, а контракт качества между разработчиком и пользователем. Любая новая фича проходит проверку на соответствие этим бюджетам.
