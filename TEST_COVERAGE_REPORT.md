# Отчет по покрытию тестами

Этот проект нацелен на обеспечение высокого качества кода через Unit-тестирование с целевым показателем покрытия **≥70%** для всех измененных модулей.

## 1. Текущий статус покрытия

На текущий момент основные компоненты логики покрыты тестами:
*   **`AutoReplyViewModel`**: ~90% (Happy Path, отмена, повторные запуски).
*   **`ChatRepository`**: Базовое покрытие логики получения данных и обработки исключений (`ChatNotFoundException`).

## 2. Анализ пропущенных сценариев и план их закрытия

Для достижения целевых показателей сформирован следующий план:

| Модуль / Класс | Пропущенные сценарии | Приоритет |
| :--- | :--- | :--- |
| **`AuthViewModel`** | Валидация номера, смена страны, обработка ошибок API (401, 500). | **Высокий** |
| **`ChatRepository`** | Параллельная обработка (ExecutorService), экспорт истории. | **Высокий** |
| **`SecurityManager`** | Шифрование/дешифрование пустых строк, работа при отсутствии ключа. | Средний |
| **`UserDataStore`** | Сохранение/чтение профиля, обработка корутинных исключений. | Средний |

## 3. Визуализация (Jacoco)

Для генерации подробного отчета используется Jacoco.

**Команда для запуска:** 
`./gradlew testDebugUnitTestCoverage`

**Путь к HTML-отчету:**
`app/build/reports/jacoco/testDebugUnitTestCoverage/html/index.html`

Отчет позволяет наглядно увидеть пропущенные ветки (branches) и строки кода, требующие дополнительного тестирования.
